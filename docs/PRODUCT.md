# CommitPush 제품 정의

## 1. 제품 정의

CommitPush는 작업 내용을 "커밋" 단위로 기록하고 관리하는 메모 기반 시스템이다.

단순 메모앱이 아니라,
생각과 업무를 버전관리하는 시스템을 지향한다.

하루의 작업을 코드 커밋처럼 기록하고,
카테고리 기반으로 구조화하며,
시간의 흐름에 따라 실행 과정을 시각화한다.

"당신의 작업을 커밋처럼 기록하고,
당신의 실행력을 시각화하는 시스템"

일반적인 메모앱과 달리,
노트의 변경과 진행 과정을 시간 흐름에 따라 누적하고 관리하는 구조를 가진다.

## 2. 기본 구조

📌 메모는 노트 단위로 존재한다.

📆 기록은 일자별 로그(Commit) 형태로 쌓인다.

🚀 저장은 Commit Push 버튼을 통해 이루어진다.

🗂 저장 시 로그는 사전 정의된 카테고리로 분류된다.

📊 각 노트는 히스토리 보드(타임라인 또는 칸반 형태) 로 확인할 수 있다.

## 3. 핵심 개념

노트는 하나의 작업 단위다.

커밋은 해당 노트의 시간 축 기록이다.

카테고리는 커밋을 구조화하는 기준이다.

모든 기록은 누적되며 삭제 대신 이력으로 남는다.

CommitPush는 단순 기록을 넘어서,
작업의 변화와 흐름을 관리하는 구조를 제공한다.

## 4. 왜 필요한가?

대부분의 메모앱은 기록은 가능하지만
그 기록이 어떻게 발전했는지 보여주지 않는다.

CommitPush는 다음을 해결한다:

작업의 누적 히스토리 추적

카테고리 기반 실행 분석

노트의 발전 과정 시각화

일 단위 실행 증빙

실행 데이터 축적

이 앱은 단순 기록 도구가 아니라,
실행 과정을 추적하고 관리하는 시스템이다.

## 5. 사용자 흐름

사용자가 노트를 생성한다.

해당 노트에 작업 내용을 작성한다.

카테고리를 선택한다.

Commit Push 버튼을 누른다.

기록이 저장되고 노트에 이력이 추가된다.

노트 상세 화면에서 커밋 히스토리를 확인한다.

**PushMind**: Free 플랜은 **구조적 질문**(최근 커밋·노트 개수 등)만 가능하고, Pro/Team 플랜은 **노트·커밋 내용 검색**과 **구조적 조회**를 함께 사용해 답변하며, 참고한 노트·커밋 출처를 표시한다.

## 6. 제품 철학

기록은 가볍게

저장은 구조적으로

분석은 자동으로

실행은 데이터로 남는다

모든 작업은 하나의 커밋이다.
모든 커밋은 실행의 흔적이다.

## 7. 데이터 구조 개념

한 명의 사용자(User)는 여러 개의 노트(Notes)를 가진다.

한 개의 노트는 여러 개의 커밋(Commits)을 가진다.

커밋은 생성 시점과 카테고리 정보를 포함한다. 첨부는 구글 드라이브 링크 등 참조만 저장하며, 파일 업로드는 하지 않는다.
